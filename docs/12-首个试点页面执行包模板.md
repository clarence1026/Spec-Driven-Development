# 首个试点页面执行包模板

## 作用

本模板用于 AI 前端交付工程规范的首个单页试点。

重点不是一次性做全项目，而是为一个真实页面准备一套可直接执行、可 review、可回写的最小闭环材料。

## 先决定采用哪种模式

试点页面先判断使用哪种模式：

- 标准模式：需要完整 `Feature Brief`、`Design Contract`、`Page Spec`
- 轻量模式：允许使用最小任务规格或 `Page Spec patch`
- 变更模式：先从 `Change Request` 开始，判断影响层级

## 执行包至少应包含什么

不管使用哪种模式，执行包都应至少包含以下内容：

1. 原始输入来源
2. 当前需求理解结果
3. 当前页面规则表达
4. 当前行为规格表达
5. `Review Checklist`
6. `Implementation Record`
7. 证据材料目录

## 推荐目录结构

### 标准模式

```text
pilot/
  order-list/
    00-raw-input.md
    01-feature-brief.md
    02-design-contract.md
    03-page-spec.json
    04-review-checklist.md
    05-implementation-record.md
    evidence/
```

### 轻量模式

```text
pilot/
  order-list-quick-change/
    00-raw-input.md
    01-task-summary.md
    02-page-spec-patch.json
    03-review-checklist.md
    04-implementation-record.md
    evidence/
```

建议规则：

- 页面目录使用 `kebab-case`
- 文档按执行顺序编号
- 截图、录屏、对照材料统一放在 `evidence/`
- 如果后续形成可复用模式，再把最终版本回写到正式文档或资产目录

```md
- 页面名称：

- 页面类型：
- 执行模式：<标准模式 / 轻量模式 / 变更模式>
- 需求确认人：
- 页面规则确认人：
- 实现与回写负责人：
- 交付裁决人：

```

```md
- <验收提示 1>
```

## 三、标准模式最小文件要求

- `01-feature-brief.md`
- `02-design-contract.md`
- `03-page-spec.json`
- `04-review-checklist.md`
- `05-implementation-record.md`

## 四、轻量模式最小文件要求

- `01-task-summary.md`
- `02-page-spec-patch.json` 或最小任务规格
- `03-review-checklist.md`
- `04-implementation-record.md`

## 五、首轮试点执行顺序

1. 先确认页面采用哪种模式
2. 整理原始输入
3. 形成需求理解结果
4. 形成页面规则表达
5. 形成行为规格表达
6. AI 执行器 / 实现负责人基于当前行为规格实现页面
7. 使用 `Review Checklist` 进行 review
8. 回写 `Implementation Record` 并判断资产候选

## 执行顺序图

```mermaid
flowchart LR
    A[原始输入] --> B[需求理解结果]
    B --> C[页面规则表达]
    C --> D[行为规格表达]
    D --> E[Frontend 实现]
    E --> F[Review Checklist]
    F --> G[Implementation Record]
    G --> H[资产候选]
```

## 六、首轮试点完成标准

首轮试点完成，至少意味着：

- 关键事实表达齐备
- 页面已完成一轮 review
- 偏差已被记录
- 至少产出 1 项资产候选
- 能明确下一轮工具化优先级
