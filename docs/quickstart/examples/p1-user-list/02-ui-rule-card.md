# UI 页面规则确认卡

## 页面信息
- 页面名称：用户列表页
- 所属任务：用户运营后台一期
- 对应 Figma / 原型：用户列表页设计稿
- 对应 PRD / 需求：用户查询与筛选

## 页面目标
- 这页主要帮助用户完成什么：快速查找目标用户并查看基础状态
- 本轮改动重点是什么：状态筛选、空态与错误态清晰可见

## 页面结构
- 主区域 1：页面头部
- 主区域 2：筛选栏
- 主区域 3：列表区
- 哪个区域是主操作区：筛选栏 + 列表区

## 关键状态
- 默认态：展示列表数据
- 加载态：列表骨架或 loading
- 空态：展示无数据提示
- 错误态：展示错误提示与重试
- 禁用 / 权限态：无详情权限时不显示跳转入口

## 关键交互
- 搜索：输入关键词 -> 更新列表 -> 保留关键词
- 状态筛选：切换状态 -> 更新列表 -> 当前状态高亮
- 点击行：点击用户行 -> 进入详情 -> 无权限时提示不可访问

## 展示规则
- 哪些字段必须展示：用户名、状态、创建时间
- 哪些字段允许折叠 / 截断：备注字段
- 哪些内容有格式限制：时间统一格式化
- 哪些信息有优先级要求：状态高于备注

## 边界与例外
- 无数据时：显示空态文案和返回筛选建议
- 异常失败时：显示错误提示和重试
- 超长内容时：备注截断
- 特殊角色 / 权限差异：无权限不显示详情入口

## 适配要求
- PC：筛选栏横向展示
- Pad：筛选栏可换行
- Mobile：筛选栏纵向堆叠，列表字段缩减

## 设计系统依赖
- 复用组件：表格、标签、按钮、输入框
- 复用 token / theme：后台通用列表页 token
- 本页特例：无

## 本轮明确不做
- 批量操作
