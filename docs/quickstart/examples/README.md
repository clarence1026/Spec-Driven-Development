# Examples Directory

当前目录中的案例主要保留为：

- 历史逻辑样本
- 帮助理解页面闭环由哪些逻辑部分组成
- 为未来 schema / workflow 抽象提供参考

当前默认执行路径已经统一到：

- `docs/superpowers/specs/`
- `docs/superpowers/plans/`

因此，这里的案例不再代表当前页面的默认启动方式。  
当前更推荐的做法是：

1. 先创建 1 份 spec 主文件
2. 再创建 1 份 plan 主文件
3. 由 AI 起草初稿
4. 由责任人确认并推进实现

如果需要理解过去的逻辑拆分方式，可以继续参考：

- `docs/quickstart/examples/p1-user-list/`
