# Review Marketing Checklist（营销首页检查清单）

## 适用范围

这份 Checklist 适用于：

- 官网首页
- 产品介绍首页
- 营销落地页首页

## 使用方式

这份 Checklist 用于 review 时直接对照检查，不替代 spec。

推荐做法：

1. 先对照页面 spec
2. 再按下面检查项逐条确认
3. 将偏差记录回 spec 的 `Writeback / Asset Candidates`

## 结构检查

- Hero、价值点区、能力区、流程区、FAQ、底部 CTA 是否完整
- 页面信息顺序是否清楚
- 各 section 主次关系是否合理

## 表达检查

- Hero 是否在首屏讲清楚产品是谁、解决什么问题、下一步做什么
- 主 CTA / 次 CTA 是否清楚区分
- 核心价值点是否控制在可快速理解的数量内
- FAQ 是否保持首页应有的信息密度

## 交互检查

- 主 CTA 是否可点击
- 次 CTA 是否可点击
- FAQ 展开 / 收起是否可用
- 能力模块跳转是否可用

## 视觉检查

- 标题层级是否清晰
- section 间距是否一致
- 卡片、按钮、标签样式是否统一
- 是否优先复用已有 theme / token，而不是局部临时拼接

## 多端检查

- `PC` 端是否有清晰首屏层级
- `Pad` 是否保留主要结构并压缩布局
- `Mobile` 是否仍能快速浏览、理解和点击 CTA

## 回写检查

- 是否记录了实现偏差
- 是否记录了被接受的例外
- 是否判断了 Hero / CTA / spacing / section 规则是否值得沉淀为共享资产

## 一句话判断

如果一个营销首页在结构、表达、交互、视觉、多端和回写这 6 个方面都能通过检查，才算真正具备复用价值。
