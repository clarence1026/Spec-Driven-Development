# Review Admin Checklist（后台列表页检查清单）

## 适用范围

这份 Checklist 适用于：

- 标准后台表格列表页
- 带筛选、分页、行级操作的列表页
- `PC / Pad / Mobile` 三端表达

## 使用方式

这份 Checklist 用于 review 时直接对照检查，不替代 spec。

推荐做法：

1. 先对照页面 spec
2. 再按下面检查项逐条确认
3. 将偏差记录回 spec 的 `Writeback / Asset Candidates`

## 结构检查

- 页面是否包含标题区、筛选区、列表区、分页区
- 区块顺序是否清楚，信息层级是否合理
- 表格列和行级操作是否与 spec 一致

## 状态检查

- `loading` 是否可见
- `empty` 是否可见
- `error` 是否可见
- `no-permission` 是否按规则处理

## 交互检查

- 搜索是否可用
- 筛选是否可用
- 分页是否可用
- 行级操作是否可用
- 跳转或按钮行为是否与 spec 一致

## 数据与展示检查

- 字段顺序是否正确
- 格式化展示是否正确
- 脱敏逻辑是否正确
- 状态标签是否正确

## 权限检查

- 页面访问权限是否符合预期
- 行级操作是否按权限展示
- 不应可见的操作是否被隐藏或禁用

## 多端检查

- `PC` 是否保留完整结构
- `Pad` 是否保持主要结构和操作
- `Mobile` 是否保留核心筛选、核心字段和核心操作

## 回写检查

- 是否记录了实现偏差
- 是否记录了被接受的例外
- 是否判断了本轮是否产生共享资产候选

## 一句话判断

如果一个后台列表页在结构、状态、交互、权限、多端和回写这 6 个方面都能通过检查，才算真正跑通闭环。
