# 资产目录

## 作用

本目录用于统一登记 AI 前端交付体系中的共享资产。

资产的意义不是“归档”，而是“让下次任务可以直接消费”。

## 资产分层

- `L1` 项目内资产：只在单个项目内稳定复用
- `L2` 团队共享资产：同一团队的多个项目可复用
- `L3` 通用共享资产：跨项目、跨团队可复用

## 资产类型

建议至少分为以下六类：

- `输入资产`：原始输入模板、需求理解模板
- `规则资产`：`Design Contract` 模板、规则模式
- `规格资产`：`Page Spec` schema、patch 模式
- `代码资产`：组件、hooks、section 模板、页面骨架
- `验证资产`：review checklist、validator 规则
- `案例资产`：闭环案例、反例、试点记录

## 使用规则

1. 先在 `Implementation Record` 中登记候选资产
2. 再判断是否升级
3. 升级后登记到本目录
4. 同时明确消费入口：谁会在下次任务里用到它

## 资产建议存放位置

| 资产类型 | 推荐落点 |
| --- | --- |
| 输入资产 / 规则资产 / 规格资产 / 验证资产 | 规范仓库、schema 仓库、文档目录 |
| 代码资产 | 组件库、monorepo package、业务基础包 |
| 案例资产 | 文档仓库、知识库、案例目录 |

## 推荐登记字段

建议统一记录：

- 资产名称
- 层级
- 类型
- 来源
- 当前状态
- 维护人
- 位置
- 消费入口
- 适用边界

## 维护动作

每次需求结束后至少完成一次快速判断：

1. 是否形成候选资产
2. 候选资产属于哪一层
3. 是否要升级
4. 维护人是谁
5. 下次由谁、通过什么入口消费
