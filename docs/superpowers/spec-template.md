# Superpowers Spec Template

# <Page Name> Spec

## 1. Task Context
### Goal
- 页面要解决的问题

### Scope
- 本次范围内内容
- 本次不做内容

### Users / Roles
- 主要用户
- 权限假设

### Input Sources
- PRD / Issue
- Figma / 标注 / 评论
- 历史页面 / 共享资产

### Acceptance
- 验收人
- 验收重点

## 2. UI Rules
### Page Structure
- 页面区块

### States
- loading / ready / empty / error / no-permission（如有）

### Interactions
- 关键交互链路

### Display Rules
- 展示 / 显隐 / 格式化规则

### Visual / Token Rules
- 优先写语义化 token / variable 名称
- 如已有 Figma Variables，写清变量名
- 如项目代码已有 token，写清对应 token 名称
- 如还没有共享 token，暂记临时色值 / 间距值，并标记为候选资产

### Responsive Rules
- PC / Pad / Mobile 表达方式

### Open Questions
- 待确认问题

## 3. Page Spec
### Layout
- 页面布局说明

### Data / Fields
- 核心字段与展示要求

### Interaction Flow
- 用户动作 -> 页面反馈 -> 结果

### Constraints
- 业务约束 / 技术约束 / 本轮边界

### Theme / Token Usage
- 已复用的 theme / token / 变量
- 当前页面新增的临时色值 / 间距 / 字重 / 圆角
- 哪些值只属于当前页面，哪些值可能升级为共享 token

### Reuse
- 复用的 pattern / rule / theme / token / kit

## 4. Review Focus
- 实现时最容易偏掉的点
- review 必查点

## 5. Writeback / Asset Candidates
### Implementation Notes
- 实现差异
- 被接受的偏差

### Decisions
- 新增裁决与例外处理

### Asset Candidates
- pattern / rule / theme / token / kit / prompt / workflow 候选

## 6. Status
- Draft / Confirmed / In Progress / Done
