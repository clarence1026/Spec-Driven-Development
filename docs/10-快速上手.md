# 快速上手

## 15 分钟先记住什么

这套体系的核心不是多写文档，而是让任意需求都能稳定进入一条 AI 可执行、可验证、可回写的工程链路。

你只需要先记住四件事：

1. 任务先选模式：标准模式、轻量模式或变更模式
2. 进入实现前，一定要形成当前规则与当前规格
3. 关键结论要有默认确认人
4. 做完一定要回写并判断资产候选

## 常用术语说明

- `Feature Brief` = 功能简报
- `Design Contract` = 设计契约
- `Page Spec` = 页面规格
- `Implementation Record` = 实现记录
- `Review Checklist` = 评审清单

## 默认流程

标准模式：

`需求输入 -> Feature Brief -> Design Contract -> Page Spec -> Frontend -> Review -> Implementation Record -> Asset`

轻量模式：

`需求输入 -> 最小任务规格 / Page Spec patch -> Frontend -> Review -> Implementation Record -> Asset`

变更模式：

`Change Request -> 更新当前规则 / 当前规格 -> Frontend -> Review -> Implementation Record`

## 第一周最小动作

- 选一个真实页面
- 判断采用标准模式、轻量模式还是变更模式
- 整理原始输入
- 形成当前需求理解结果
- 形成当前页面规则表达
- 形成当前行为规格表达
- 约定 review 与回写方式
